fielddata와 doc_value 이해하기

doc_value

대부분의 필드들은 기본적으로 검색이 가능하도록 색인된다.

역색인을 통해 쿼리는 검색단어를 찾고 그 단어를 포함하고 있는 문서들의 리스트에 즉시 접근이 가능하도록 하다.

하지만 정렬,집계 그리고 스크립트에서 필드값들에 접급하는 것은 검색과는 다른 접근 방식을 요구한다.

단어를 찾아서 문서를 찾는 대신에, 문서를 찾아서 필드에 포함된 용어를 찾을 수 있어야한다.

Doc values는 색인할때 빌드되는 디스트 기반 데이터 구조체이다.

이 데이터 접근 방식 패턴을 가능하게 한다. (문서를 찾아서 필드에 포함된 용어를 찾는 방식?)

_source 부분에 같은 값을들 저장하지만 정렬,집계에 더 효율적인 컬럼지향방식으로 저장한다.

Doc values는 대부분의 필드 타입을 지원한다. (text,annotated_text 필드타입은 제외)

문서 값을 지원하는 모든 필드는 기본적으로 활성화되어 있다.

만약 정렬이나 집계 그리고 스크립트 상에서 필드 값에 접근하는 것이 필요하지 않다면 디스크용량을 절약하기위해 비활성화 할 수 있다.

fielddata

대부분의 필드들은 기본적으로 검색이 가능하도록 색인된다

하지만 정렬,집계 그리고 스크립트에서 필드값들에 접급하는 것은 검색과는 다른 접근 방식을 요구한다.

검색은 '어느 문서에 특정 단어를 포함하고있는가?'에 대한 답을 필요로하지만 정렬이나 집계는 '이 문서에서 필드 값은 무엇인가?'와 같은다른 질문을 필요로한다.

"Which documents contain this term?

"What is the value of this field for this document?"

대부분의 필드들은 색인할때 doc_values를 통해 다른 접근방식('이 문서에서 필드 값은 무엇인가?' =문서를 찾아서 필드에 포함된 용어를 찾는 방식)을 사용할수 있다. 하지만 doc_values는 text 필드를 지원하지 않는다.

대신에 text 필드는 검색 질의할때 in-memory 데이터 구조를 가진 fielddata를 사용할 수 있다. 이 데이터 구조는 필드가 집계, 정렬 또는 스크립트에 처음 사용될 때 필요에 따라 구축된다.

디스크에서 각 세크먼트에 대한 전체 역색인을 읽어들이고, 단어와 문서관계를 반전시킨 다음, 결과를 JVM heap 메모리상에서 정렬함으로써 구축이된다.

  • 단어와 문서관계를 반전시킨다는게 document를 키로, field 정보를 값으로 담은 데이터 구조로 만든다는 것 같음.

  • 기존 역색인 구조는 단어를 키로, 문서를 값으로 담은 데이터 구조니까.

기본적으로 text 필드에 대해 fielddata는 비활성화 되어있다.

fielddata는 높은 카디널리티 text 필드를 로드 할 때 많은 힙메모리를 소모한다.

fielddata가 힙메모리상에 로딩되면 계속 세그먼트 수명동안 유지된다. -> 비용이 높은 프로세스가 된다.

또한, fielddata로딩하는 것은 비용이 많이 드는 작업이므로 지연시간이 발생 할 수 있다.

그래서 이러한 이유때문에 기본적으로 비활성화되어있다.

집계,정렬,스크립트사용에서 text필드를 사용하면 다음과 같은 exception을 보게될것이다.

Fielddata is disabled on text fields by default.  Set `fielddata=true` on
[`your_field_name`] in order to load  fielddata in memory by uninverting the
inverted index. Note that this can however use significant memory.

fielddata 활성화하기전에...

fielddata활성화하기전에, 집계,정렬,스크립트사용에 text필드를 사용하는 이유를 고려해라.

일반적으로 그렇게하는 것은 의미가 없다.(?)

  • text 필드는 New York을 색인하기 전에 new나 york로 검색이 가능하도록 분석된다.

  • terms 집계에서는 New York가 담긴 버킷이 아닌 new 버킷과 york 버킷이 결과로 나올 것이다.

  • 그로므로, 전체 텍스트 검색을위한 text 필드와 집계를 위해 doc_values가 활성화 된 분석되지 않은 keyword 필드가 있어야한다.

    curl -X PUT "localhost:9200/my_index?pretty" -H 'Content-Type: application/json' -d'
{
  "mappings": {
    "properties": {
      "my_field": { 
        "type": "text",
        "fields": {
          "keyword": { 
            "type": "keyword"
          }
        }
      }
    }
  }
}
'

fielddata_frequency_filter

fielddata 필터링을 통해 메모리상에서 로딩되는 단어 수를 줄일수 있다. 그러면 메모리 사용량도 줄어든다.

단어들은 빈도수에 의해 필터링된다.
